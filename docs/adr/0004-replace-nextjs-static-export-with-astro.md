# ADR 0004: Replace Next.js Static Export with Astro

## Status

Accepted (2026-07-21)

Supersedes: [ADR 0001](0001-use-nextjs-static-export.md)

## Context

The site is a static personal portfolio and blog with content stored as Markdown files. ADR 0001 selected Next.js with `output: 'export'`, but the site only uses routing, metadata generation, sitemap, and robots — features Astro provides natively with less overhead.

The migration should preserve:
- Content in `content/pages/` (ADR 0002)
- Layout and section registries (ADR 0003)
- All public routes, trailing-slash URLs, and static behavior

## Decision

Replace Next.js with Astro as the static site generator. Keep React for existing TSX components via `@astrojs/react`. Do not migrate content to Astro Content Collections in this iteration.

## Consequences

- **Framework is now Astro 7**, with static output to `out/`.
- **Routing is handled by Astro** via `src/pages/index.astro`, `src/pages/[...slug].astro`, and `src/pages/404.astro`.
- **Metadata is rendered client-side** through `src/components/Seo.astro`, reading from page frontmatter.
- **Sitemap is generated by `@astrojs/sitemap`** integration.
- **Robots.txt is a static file** in `public/`.
- **Fonts use `@fontsource-variable/karla`** instead of `next/font/google`.
- **Images use a StaticImage wrapper** instead of `next/image`.
- **Links use plain `<a>` tags** instead of `next/link`.
- **Client interactivity** (menu toggle, video embeds) is handled by small vanilla scripts in `src/scripts/` and `public/scripts/`.
- **TypeScript checking** moved from `tsc --noEmit` to `astro check`.
- Next.js dependencies, `next.config.js`, `next-env.d.ts`, and `src/app/` are removed.
- Build output (`out/`) and public URLs remain unchanged.
